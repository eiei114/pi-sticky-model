# Security Policy

## Supported versions

Only the latest published version receives security fixes.

## Reporting a vulnerability

Report vulnerabilities through [GitHub Security Advisories](https://github.com/eiei114/pi-sticky-model/security/advisories/new).

You can also open a [private security issue](https://github.com/eiei114/pi-sticky-model/security/policy).

Please include:

- Affected version
- Impact
- Reproduction steps
- Suggested fix, if known

## Pi package security note

Pi packages can execute code with local user permissions. Review installed packages and avoid running untrusted extensions.
